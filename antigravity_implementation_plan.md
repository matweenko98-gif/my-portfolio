# Документ: Инструкция и пошаговый план реализации в Antigravity (antigravity_implementation_plan.md)

## 1. Пошаговый технический план сборки

### Шаг 1: Создание глобальной двухколоночной сетки (Global Layout)
1. Создай корневой контейнер (`Root Container`) для всего приложения.
2. Внутри корневого контейнера инициализируй двухколоночную структуру (используя `Flexbox` со свойством `row` или `Grid Layout`).
3. **Левая колонка (Sidebar):**
   * Задай фиксированную ширину (например, `width: 25vw` или фиксированные `320px`).
   * Установи параметр `position: fixed`, верхняя точка `top: 0`, левая точка `left: 0`.
   * Задай высоту `height: 100vh` и свойство `overflow: hidden`, чтобы она не скроллилась сама по себе.
4. **Правая колонка (Content Area):**
   * Задай ширину на оставшееся пространство экрана (например, `width: 75vw` или `flex-grow: 1`).
   * Сделай её отступ слева равным ширине сайдбара (`margin-left: 320px`), чтобы контент не перекрывался зафиксированной панелью.
   * Установи параметры скролла: `overflow-y: scroll`, `height: 100vh`.
5. **Адаптивность (Mobile Breakpoint < 1024px):**
   * Настрой медиа-запрос: левая колонка переходит в стандартный `position: relative`, `width: 100%`, `height: auto`. Меню скрывается в бургер. Маргин правой колонки `margin-left` сбрасывается на `0`. Контент выстраивается в единую вертикальную ленту.

### Шаг 2: Верстка и наполнение Левой панели (Sidebar)
1. Внутри левого контейнера размести вертикальный стек элементов (`Flex-direction: column`).
2. Добавь верхний блок профиля: контейнер под аватар (круглый или со скруглением `rounded-full` серый блок-заглушка) и текстовые элементы для имени и роли из `content.md`.
3. Размести текстовый блок `ShortBio`, настроив шрифт Inter, цвет `text-zinc-500` и размер `text-sm` или `text-base`.
4. Создай компонент навигации `NavigationMenu`. Каждый пункт меню должен являться интерактивной текстовой ссылкой. Привяжи к ссылкам якорные ID секций правой части (`#hero`, `#services`, `#cases`, `#workflow`, `#contacts`).
5. В самый низ контейнера прижми блок `FooterContacts`, используя `margin-top: auto`.

### Шаг 3: Верстка каркаса правой части (Static Skeleton)
1. Внутри правой скроллящейся колонки последовательно создай 5 крупных контейнеров-секций.
2. Присвой каждому контейнеру уникальный ID: `hero`, `services`, `cases`, `workflow`, `contacts`.
3. Задай для всех секций единый стандарт внутренних вертикальных отступов (например, `padding-top: 100px`, `padding-bottom: 100px`), чтобы контент имел много свободного пространства ("воздуха").
4. Перенеси тексты из файла `content.md` в соответствующие блоки секций. Используй теги H1 для главного заголовка, H2 для заголовков секций и H3 для карточек. На этом этапе стили делай базовыми, фокусируясь на правильном расположении элементов.

### Шаг 4: Настройка динамического состояния (State Management) в секции Услуг
1. Перейди к секции `services`. Внутри сетки карточек выдели компонент `ServiceCard`.
2. Создай локальную переменную состояния для карточки — `isOpen` типа Boolean со значением по умолчанию `false`.
3. Привяжи триггер `onClick` (событие нажатия) на заголовок или всю область карточки: при клике значение `isOpen` меняется на противоположное (`!isOpen`).
4. Настрой условный рендеринг (Conditional Rendering): внутренний блок деталей карточки (где прописаны тарифы, сроки и бриф) отображается на экране только в том случае, если `isOpen === true`.

### Шаг 5: Реализация заглушек кейсов портфолио
1. В секции `cases` создай сетку `Grid` (2 колонки на десктопе, 1 на мобильном, расстояние между элементами `gap: 24px`).
2. Внутри каждой карточки кейса создай верхний прямоугольный контейнер под изображение. Задай ему фиксированную высоту (например, `height: 300px`), скругление углов `rounded-2xl` и фоновый цвет `bg-zinc-100`. По центру добавь неброский текст: "Изображение кейса (Заглушка)".
3. Под блоком заглушки размести текстовые блоки: крупную метрику, название и текстовую структуру Задача/Решение/Результат.

### Шаг 6: Интеграция анимаций и тонкая настройка UI
1. Подключи триггеры отслеживания скролла (Scroll In-view) для секций правой колонки, настроив появление согласно документу `animation_guide.md`.
2. Настрой ховеры (`hover:`) для кнопок, карточек услуг и пунктов меню навигации.
3. Протестируй макет на разных разрешениях экрана в интерфейсе Antigravity.
